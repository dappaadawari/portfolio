Stat & Progression System — Races, Leveling & Skill Points

A full RPG progression layer with weighted race and variant rolling, procedurally generated names by gender, a nonlinear 100-level XP curve, handcrafted skill point rewards, and a reactive HUD that updates live from player attributes.

The Problem

Meaningful character identity from the moment you spawn

Most games assign every player the same base stats and the same starting experience. The goal here was a system where your character feels distinct immediately — a randomly rolled race with its own stats, a procedurally generated name, and a progression path that requires real investment to develop.

Design goal: Character identity should be established on spawn and deepen over time — through race bonuses, stat allocation, and a leveling curve that rewards long-term play.
Race System

Weighted rolling with rare variants

Six races are defined with individual spawn weights (Human 0.80, Elf 0.32, Giant 0.12, down to Demon, Angel, and Undead at 0.02 each). On spawn, a cumulative probability roll determines the player's race. A second roll then picks a race variant — each race has 3–4 variants with their own sub-weights, including ultra-rare variants like Seraphim (0.01) or Sin Demon (0.01).

local function rollRace()
    local randomValue = math.random()
    local cumulativeChance = 0
    for raceName, data in pairs(races) do
        cumulativeChance = cumulativeChance + data.chance
        if randomValue <= cumulativeChance then
            local chosenVar = rollVariant(data.raceName)
            return data.raceName, chosenVar
        end
    end
end
Procedural Names

Names that match character gender

Each player is assigned a gender (Masculine/Feminine) then a name randomly selected from a curated list of 50+ names per gender — fantasy-leaning names designed to fit the game's world. The name is stored as a player attribute and displayed in the HUD. This small detail makes every character feel like a unique person rather than "Player123."

Leveling Curve

A nonlinear XP formula that scales with level

XP needed to level up uses a piecewise formula — one curve for levels 1–49, a steeper curve for 50–89, and the steepest for 90–100. This keeps early levels fast and rewarding while making the endgame a serious commitment. The formula is evaluated live so tweaking the constants changes the whole curve without touching any other code.

local function ExpNeeded(level)
    if     level < 50  then return math.floor(100  * level^1.5)
    elseif level < 90  then return math.floor(150  * level^1.7)
    else               return math.ceil (200  * level^2.0)
    end
end
Skill Points

Handcrafted rewards, not a simple formula

Skill point rewards are defined in an explicit 100-entry lookup table rather than calculated by formula. This gives full control over the reward pacing — bigger bursts in the early-mid game (levels 5–40), tapering off in the endgame, with "rest" levels at every third level that award nothing. Players don't earn skill points until level 5, giving the early game time to establish identity before build decisions begin.

HUD

Reactive UI driven by attribute signals

The HUD listens to player attribute change signals directly — GetAttributeChangedSignal for Gold, Level, Exp, SkillPoints, and Mana. Health and MaxHealth use Humanoid property signals. Every bar and label updates live without polling or RemoteEvents. The stats panel is built dynamically from the Stats folder's attributes so new stats are added automatically without touching the HUD code.

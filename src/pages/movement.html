Movement System: Sprint, Dash, Double Jump & Sliding

Server authoritative movement with sprint, directional dashing, double jump, and sliding utilizing LinearVelocity and AlignOrientation. Client plays animations and handles input.

The Problem

Syncing server & client to make responsive movement.

Roblox's default character controller is limited; no sprinting, no dashing, no sliding. The goal was a full movement system where all speed modifications and physics forces are applied server side, while animations and input remain on the client for responsiveness.

Key challenge: Making movement feel real and responsive.
Dashing

Directional dashing with diagonal support

The client reads held WASD keys persistently via InputBegan/InputEnded. When Q is pressed, the held keys are sent to the server as a string like "WA" or "WD".

The server computes the dash direction from the character's CFrame — each key in the combo contributes its axis, so diagonals work naturally. The direction string is validated (1–4 chars, only WASD) before any physics are applied. A LinearVelocity constraint is added to the HumanoidRootPart for the dash duration then destroyed.

if keys:find("W") then direction += lookVector  end
if keys:find("S") then direction -= lookVector  end
if keys:find("D") then direction += rightVector end
if keys:find("A") then direction -= rightVector end
Sliding

Physics based sliding that responds to terrain

Sliding uses a LinearVelocity for forward momentum and an AlignOrientation to tilt the character to match the surface normal so that they lean into slopes visually. A raycast fires downward every Heartbeat; if no ground is detected the slide stops automatically.

A speed multiplier starts at 1.0 and changes based on vertical position delta each frame, accelerating downhill, decelerating on flat ground, and decelerating faster going uphill. When the multiplier drops below 0.1 the slide ends naturally.

if verticalChange < 0.1 and verticalChange > -0.1 then
    currentMultiplier = math.clamp(
        currentMultiplier - slideSettings.SpeedChangeRate.Forward * dt,
        0, slideSettings.MaxMultiplier)
elseif verticalChange < 0 then -- going downhill, add speed
    currentMultiplier = math.clamp(
        currentMultiplier + slideSettings.SpeedChangeRate.Downward * dt,
        0, slideSettings.MaxMultiplier)
end
Double Jump

Mid air jump using LinearVelocity

Double jump fires a brief LinearVelocity along the Y axis for 0.12 seconds, then destroys itself. It's gated behind a FloorMaterial == Air check and a per-character cooldown attribute.
